# Changelog

## v2.1.0

### Новые возможности
- **SQLite вместо JSON** — все данные теперь хранятся в SQLite базе данных (`panel.db`). Это обеспечивает целостность данных, поддержку транзакций и параллельный доступ без блокировок.
- **Полная асинхронность** — все операции с базой данных выполняются через `aiosqlite`, что обеспечивает неблокирующую работу с БД в асинхронном цикле FastAPI.
- **Умное кеширование** — реализован TTL-кеш для данных пользователей и серверов, а также кеш SSH-запросов для списка клиентов протоколов. Это значительно снижает количество запросов к БД и SSH при частых обращениях.

### Удалено
- **Импорт пользователей** — полностью удалён функционал импорта и синхронизации пользователей (Remnawave и другие источники).
- **Remnawave Sync** — удалены все эндпоинты и настройки, связанные с синхронизацией пользователей из Remnawave.
- **data.json** — однофайловое JSON-хранилище заменено на SQLite. Файл `panel.db` создаётся автоматически при первом запуске.

### Исправления
- **Telemt «No users configured»** — исправлена ошибка, при которой Telemt-сервис не запускался из-за удаления пользователя `hello` по умолчанию при установке. Теперь пользователь `hello` сохраняется в конфигурации.
- **`_check_admin` стал async** — функция проверки прав теперь корректно работает с асинхронными вызовами БД.
- **Бэкап/восстановление** — экспорт и импорт данных теперь работает через SQLite (экспорт в JSON для скачивания, импорт из JSON с восстановлением в БД).

### Технические изменения
- Добавлена зависимость `aiosqlite>=0.19.0`
- Создан модуль `managers/db.py` — асинхронный слой работы с SQLite
- Создан модуль `managers/cache.py` — TTL-кеш для данных и SSH
- Обновлён `Dockerfile` и `docker-compose.yml`
- Обновлены тесты для работы с SQLite
- Добавлены регрессионные тесты для проверки миграции
